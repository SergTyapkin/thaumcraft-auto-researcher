# 이것을 어떻게 사용하나요?
### 사전 설정 
> _프로그램 최초 시작 후 1회 실행_
0. [릴리스](https://github.com/SergTyapk)에서 프로그램을 다운로드하세요.in/thaumcraft-auto-researcher/releases)
1. 십자선이 움직일 수 있다는 시연 및 검증. 
빨간색 점을 노란색 점으로 옮기면 됩니다.
![image](https://github.com/SergTyapkin/thaumcraft-auto-researcher/blob/master/README_images/enroll.png?raw=true)
2. 연구 테이블 인터페이스가 어디에 있는지 프로그램에 알려주어야 합니다. 
이렇게 하려면 아래 스크린샷에 표시된 것처럼 노란색 직사각형의 모서리가 테이블의 외부 둘레를 따라 이동하도록 이동해야 합니다.
![image](https://github.com/SergTyapkin/thaumcraft-auto-researcher/blob/master/README_images/find_table.png?raw=true)
3. 마법 부여대 내부에서 상호 작용 버튼이 어디에 있는지 더 자세히 프로그램에 알려줄 필요가 있습니다.
이렇게 하려면 아래 스크린샷에 표시된 대로 모든 점을 이동하세요.
![image](https://github.com/SergTyapkin/thaumcraft-auto-researcher/blob/master/README_images/setup_controls.png?raw=true)
4. Thaumcraft 버전과 설치된 모든 애드온을 선택하세요.
![image](https://github.com/SergTyapkin/thaumcraft-auto-researcher/blob/master/README_images/setup_version_and_addons.png?raw=true) 

이 모든 단계를 완료한 후 모든 사용자 선택은 `C://users/%USER%/.ThaumcraftAutoResearcher` 폴더에 저장됩니다.
다음에 프로그램을 시작할 때 이를 수행할 필요는 없습니다. 다음 단계가 즉시 표시됩니다.
'백스페이스' 키를 누르면 언제든지 구성으로 돌아갈 수 있습니다.

### 측면 체인 해결
1. 왼쪽 상단 인벤토리 슬롯에 있는 **연구 노트**를 연구 테이블 슬롯에 넣습니다.
'Enter'를 누르면 신경망을 사용하여 현장에서 측면을 결정하는 프로세스가 시작됩니다.
![image](https://github.com/SergTyapkin/thaumcraft-auto-researcher/blob/master/README_images/prepare_to_solving_aspects.png?raw=true)
프로그램이 게시할 측면 체인을 사용하여 솔루션이 자동으로 생성됩니다.
![image](https://github.com/SergTyapkin/thaumcraft-auto-researcher/blob/master/README_images/aspects_solved.png?raw=true)

> [!팁]
> 측면 체인이 너무 크거나 갖고 있지 않은 측면을 사용하는 경우 `R`을 눌러 다시 생성하세요.
![image](https://github.com/SergTyapkin/thaumcraft-auto-researcher/blob/master/README_images/aspects_rerolled.png?raw=true)

> [!팁]
> 게임이 프로그램 인터페이스와 겹치지 않도록 게임에서 뭔가를 해야 할 경우 `Ctrl+Shift+Space`를 누른 다음
이 키 조합을 다시 누를 때까지 프로그램이 일시 중지됩니다.
![image](https://github.com/SergTyapkin/thaumcraft-auto-researcher/blob/master/README_images/program_paused.png?raw=true)

> [!팁]
> 셀 중 하나라도 잘못 정의된 경우 해당 셀을 클릭하고 실제로 어떤 셀이 되어야 하는지 선택할 수 있습니다.에이.
그 후에는 솔루션이 자동으로 재생성됩니다.
![image](https://github.com/SergTyapkin/thaumcraft-auto-researcher/blob/master/README_images/setup_table_aspects.png?raw=true)
2. **잉크 탱크에 잉크가 충분한지 확인하세요**. 그것들이 다 떨어지더라도 측면 배치 알고리즘은 멈추지 않을 것입니다.
연구 노트는 해결되지 않습니다.
그런 다음 `Enter`를 누르면 결과 체인에 따라 테이블에 측면을 배치하는 프로세스가 시작됩니다.
3. **측면 배치가 완료되면** 연구 노트가 인벤토리에 배치됩니다.
그 대신 인벤토리의 다음 항목이 테이블 위에 놓입니다.
그런 다음 프로세스가 다시 반복됩니다. 이렇게 하면 인벤토리에 쌓여 있는 수많은 노트를 차례로 해결할 수 있습니다.
![image](https://github.com/SergTyapkin/thaumcraft-auto-researcher/blob/master/README_images/next_research_putted.png?raw=true)

> [!팁]
> - 연구노트를 인벤토리에 있는 그대로 하나씩 계속해서 조사하려면 `Ctrl+Enter`를 누른 후
각 다음 메모를 공부할 때 프로그램은 'Enter' 키를 눌러 사용자의 확인을 기다리지 않고 즉시 솔루션 게시를 시작합니다.

> [!팁]
> - 측면 레이아웃 시 긴급하게 프로그램을 종료해야 하는 경우를 대비해 'Ctrl+Shift+Alt' 키 조합을 제공합니다.

# 소스에서 실행:
1. 종속성을 설치합니다.
```shell
pip install -r requirements.txt
```

2. 프로젝트 루트에서 실행합니다(`Python 3.10` 이상 필요).
```shell
python ./src/main.py
```

## 애플리케이션을 .exe 파일로 빌드
1. 종속성 및 빌더를 설치합니다.
```shell
pip install -r requirements.txt
pip install auto-py-to-exe
```

2. ***\[선택 단계]*** UPX 다운로드(최종 exe 파일의 크기 감소)
https://github.com/upx/upx/releases/


3. 프로젝트 루트에서 빌드 명령을 실행합니다(빌드를 실행할 수 있는 인터페이스가 열립니다).
```shell
auto-py-to-exe -c .\pyinstaller_configs\autoPyToExe.json
```

4. ***\[선택 단계]*** **고급** 섹션에서 `--upx-dir`(`upx.exe` 실행 파일이 있는 폴더 위치)을 지정하고 빌드를 실행합니다.
컴파일된 exe 파일은 이 디렉터리의 `output` 폴더에 나타납니다.
